# AI 群聊摘要助手｜概念方案（用户说得队）

## 一句话概述
把“没完没了的群聊”压缩为可操作的主题/决策/行动/资源四类卡片，并支持一键溯源、订阅“只看我相关”、与日历/看板双向联动，让重要信息不再被冲散、决策能沉淀、执行有闭环。

## 为什么是现在
- 群聊消息爆炸：重要信息被海量闲聊淹没，回溯成本高。
- 决策难沉淀：口头/临时确认与正式结论脱节，审计与追溯困难。
- 执行闭环弱：从聊天到任务/日程的“搬运”靠手工，易漏项、不同步。
- 协作碎片化：跨时区/多群协作，成员希望“只看我相关”而非通刷。

## 目标用户与典型场景
- 项目经理：需要将讨论快速沉淀为可审计“决策卡”，并驱动“行动卡”。
- 跨时区工程师：希望在固定时间窗口看到“只看我相关”的微摘要与变化点。
- 社群运营（极端用户）：同时管理 50+ 群，强调话题升温监控、资源去重与批处理。
- 听障分析师（边缘化用户）：依赖结构化文字、锚点回跳与键盘操作的可达性。
- 企业 IM 运营专家：关注权限/可见性/审计与与现有工具生态的双向同步。
- 家长与班主任（家校群）：作业/通知/请假/报名/成绩/家长会信息聚合；一键确认与回执统计。
- 社区居民与物业（社区/业委会/物业群）：报修工单、公告投票、停车/缴费信息；状态回写减少重复询问。
- 兴趣社群（跑步/摄影/游戏等）：活动报名、签到与成绩榜；资源卡沉淀教程与素材。
- 志愿者/活动组织者：任务分发、场地协调、应急广播；批量提醒与收尾复盘。
- 家庭与照护（家庭群/长辈照护）：购物清单、用药提醒、账单分摊；行动卡联动日历。
- 旅游/婚礼/搬家筹划：行程、分工、预算、供应商对接与注意事项沉淀。
- 小微商家与团长（拼团/二手交易群）：库存/订单/发货/售后要点与动作回写。
- 校园课程/班级/社团：作业布置、课程变更、考试安排、活动组织与报名统计。

## 核心价值主张
- 将长对话“变形”为四类卡片，信息结构化且可操作。
- 用微摘要与订阅机制减少无效打扰，突出“我相关”。
- 让决策与执行强绑定，状态可回写，形成端到端闭环。
- 可溯源、可审计、可达性友好，增强团队对信息与流程的信任。

## 核心能力（V1 范围）
### 四类卡片
- 主题卡（Topic）：话题聚合、热度/参与人/时间窗、版本对比、合并/拆分、锚点溯源。
- 决策卡（Decision）：结论、理由、约束/适用、异议摘要、责任人/截止、回滚提醒、锚点溯源。
- 行动卡（Action）：任务、负责人、截止、依赖、状态、外链（看板/日历），完成/改期回写。
- 资源卡（Resource）：去重与版本化、标签、预览、可信来源标记、跨群汇总与批处理。

### 订阅与推送
- 订阅维度：我相关（责任/提及/依赖）、关注主题、关键词、角色/身份域（家长/老师/社区管理员/志愿者负责人/团长等）。
- 推送形态：15–30 秒微摘要卡；静默时段/频率上限；日终 Digest 汇总。
- 触发条件：主题升温、决策变更、与我相关的行动变更、资源更新。

### 溯源与审计
- 任一要点一跳回原消息片段（锚点高亮）+上下文窗口。
- 审计轨迹：更改人、时间、变更内容与原因；可导出。

### 纠错与学习
- 一键纠正主题归类/要点/术语误判；系统学习本群“黑话/缩写”，持续提升准确度。

### 多工具联动与回写
- 日历：行动卡创建/变更同步日历事件；完成/改期回写。
- 看板/多维表：任务双向同步，标签/状态映射，冲突合并策略。
- 文档/公告：决策卡一键同步，保留锚点回链与版本号。

## 关键用户旅程（示例）
1) 讨论升温 → 自动生成“登录页 A/B 测试”主题卡。
2) 达成共识“采用版本B、周五上线” → 生成决策卡（含理由/约束/责任人/截止）。
3) 自动抽取行动卡（更新埋点、回滚预案等）→ 一键推送到看板与日历。
4) 相关链接与文件形成资源卡 → 去重/版本化、跨群复用。
5) 进展与结果回写群聊摘要 → 审计轨迹完整可查。

## 生活场景旅程（示例）
- 家校群（作业与通知）：老师在群里发布本周作业与周五家长会 → 生成主题卡；确认“周五 19:00 线上家长会” → 决策卡（含链接与注意事项）→ 行动卡（家长确认出席、请假登记）推送到日历 → 资源卡沉淀作业文档与报名表；家长确认/变更自动回写统计。
- 社区报修群（电梯维修）：多位住户反馈电梯故障 → 主题卡升温并标注楼栋/时间窗 → 物业确认“周三 10:00 到场检修” → 决策卡；行动卡（派单/维保到场/复测）分配并联动工单系统；维修报告与注意事项作为资源卡归档并公告回写。

## 信息架构与对象模型（摘要）
- Topic：id、title、status、metrics{heat,participants,timeWindow}、highlights[≤3]、anchors[]、ops{merge,split,compare}。
- Decision：id、conclusion、rationale、constraints、objections、owner、due、rollback、anchors[]、ops{announce,linkDoc}。
- Action：id、title、assignee、due、deps[]、status、externalLinks{board,calendar}、ops{complete,reschedule,mention}。
- Resource：id、sourceAnchor、preview、tags[]、version、fingerprint、ops{dedupe,merge,export}。

## MVP 与不做清单
- MVP 做：主题聚类（近实时）、1–3 条可编辑要点、锚点溯源；常见行动抽取并推送到日历/看板；“只看我相关/关键词/关注主题”订阅；基础可见性与更改日志。
- 暂不做：复杂审批流、跨企业级 SSO/合规一体化、深定制报表、自动化治理策略引擎（留待后续）。

## 技术路线（草案）
- 接入层：IM 平台 webhook/bot/导出 API；消息清洗与多媒体转写。
- 语义管道：embedding + 主题聚类；多轮摘要（微摘要→要点→长摘要）；动作/实体抽取（人名/动词/日期/依赖）。
- 溯源锚点：消息 id + 时间戳 + 片段定位；上下文窗口生成。
- 联动集成：Calendar/Boards/Sheets 双向同步；冲突合并与重放策略。
- 隐私合规：可见性域、最小必要处理、到期清理、操作留痕；家校场景下对未成年人信息最小化与“仅本孩子相关可见”的教师审核开关。

## 可达性与无障碍
- 键盘优先操作、焦点可见、结构化分块；语义标题/列表；色彩对比与可读字体。

## 度量指标（北极星与支持性指标）
- 北极星：因遗漏而导致的返工/延误降低比例；决策/行动从群聊到落地的闭环率。
- 支持性：
  - 订阅命中率、误报/漏报率、纠错采纳率。
  - 主题回溯时间中位数；锚点点击到原文的跳出率。
  - 行动卡自动抽取覆盖率/准确率；联动回写成功率。
  - 生活场景：家校通知触达与家长确认率；社区报修闭环时长；活动报名转化率。

## 风险与对策
- 语义误判：提供显著的纠错入口与学习闭环；保留原文溯源。
- 干扰过多：细粒度订阅与静默窗口、频率上限、Digest。
- 采纳阻力：从最小闭环开始（主题/决策/行动），对接少量刚需工具；保留人工兜底与退出机制。

## 竞品与差异（简）
- 竞品：IM 置顶/公告、机器人关键词、第三方看板/日历、通用摘要插件。
- 差异：以“可操作卡片+一跳溯源+双向回写”为核心闭环，而非只做信息通知或被动归档。

## 里程碑（建议）
- 第 1–2 周：接入与聚类/微摘要 MVP、手动纠错。
- 第 3–4 周：决策/行动卡联动日历与看板，状态回写。
- 第 5–6 周：订阅策略、Digest、术语学习与可见性基础。
